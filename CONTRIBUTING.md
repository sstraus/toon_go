# Contributing

Thanks for your interest in contributing to TOON Go! We welcome issues, feature
requests, and pull requests. To keep the project healthy, please follow these
guidelines.

## Getting Started
- Fork the repository and clone your fork locally.
- Install Go 1.22 or newer.
- Run `go test ./...` to ensure the test suite passes before making changes.

## Development Workflow
- Create a feature branch for your work.
- Add tests for any bug fixes or new features.
- Run `go test ./...` and ensure the CI workflow passes before submitting a PR.
- Update documentation (README, CHANGELOG) when behavior changes.

## Pull Requests
- Describe the problem and solution clearly in the PR description.
- Reference related issues and link to any relevant discussions.
- Expect maintainers to request changes; review iterations are normal.
